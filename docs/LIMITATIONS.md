# Current limitations

This release supports controlled fictional workflows and does not claim arbitrary-site compatibility. It is not approved for real sensitive accounts, real documents, OTPs, payments, financial transactions, medical-record changes, or government submissions. It does not replace professional security, privacy, accessibility, legal, or human-factors review and makes no worldwide novelty claim.

The collaboration model now supports cross-browser synthetic sessions through capability-checked Supabase RPC functions, but it does not provide named user accounts, recovery, key rotation, organisation controls, or durable encrypted user-document storage. The extension is restricted to TrustMode controlled portal routes. Moving beyond this boundary requires authenticated identity, abuse controls, observability, incident response, audited portal adapters, and independent assessment.
